## Contributing

Contributions are what make the community such an amazing place to learn, inspire, and create. **Any contributions you make are greatly appreciated.**

We follow a [Git Flow](https://nvie.com/posts/a-successful-git-branching-model/) workflow with [Conventional Commits](https://www.conventionalcommits.org):

1. Fork the [repository](https://github.com/2CodersStudio/awesome_2coders).
2. Create a feature branch:
   ```bash
   git checkout -b feature/your-resource
   ```
3. Add your resource under the appropriate category with a short, one-line description. Add the 🔹 marker only if it is part of the 2Coders stack.
4. Commit your changes with a conventional message:
   ```bash
   git commit -m 'docs: add a new awesome resource'
   ```
5. Push the branch:
   ```bash
   git push origin feature/your-resource
   ```
6. Open a Pull Request describing the resource and why it belongs here.

**Guidelines:**

- **Quality over quantity** – we prefer a tightly curated list of genuinely valuable links.
- **Short descriptions** – one concise line per entry.
- **No duplicates** – check the list before adding.
- **Working links only** – verify the URL resolves.
- **Stay awesome** – if it doesn't inspire or help, it may not fit.
